# BuildOS — QA и чек-листы этапов (v0)

## Назначение
QA-модуль фиксирует качество выполнения работ и служит:
- доказательством для клиента
- инструментом контроля для владельца
- основой для гарантий

---

## 1) Общая логика QA
- Каждый этап имеет чек-лист
- Чек-лист обязателен для закрытия этапа
- Есть статусы: OK / Defect / Rework

---

## 2) Пример чек-листов по этапам

### Демонтаж
- Все конструкции демонтированы
- Мусор вывезен
- Повреждений несущих нет

### Стяжка пола
- Ровность (допуск ±2 мм)
- Отсутствие трещин
- Выдержано время высыхания

### Электрика
- Кол-во точек соответствует проекту
- Кабель нужного сечения
- Фото распределительного щита

### Плитка
- Швы ровные
- Нет пустот
- Совпадение рисунка

### Покраска
- Равномерный слой
- Нет потёков
- Цвет соответствует ТЗ

---

## 3) Статусы приёмки
- Не начато
- На проверке
- Принято
- Принято с замечаниями
- Возврат на доработку

---

## 4) Роли в QA
- Worker — загружает фото
- Foreman — инициирует приёмку
- Inspector — утверждает
- Client — просматривает

---

## 5) История и гарантия
- Все дефекты сохраняются
- Фиксация даты и ответственного
- Используется в гарантийных спорах

---

## 6) Killer-feature
- Клиент видит не «процесс», а **качество**
- Меньше конфликтов
- Повышение доверия
