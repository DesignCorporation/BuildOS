# BuildOS — Админ-панель оркестратора (Operator Console) v0

## 1) Зачем отдельная админка
Отдельная панель для команды BuildOS (вашего SaaS), чтобы:
- видеть все компании (тенанты), планы, статус оплат
- быстро помогать клиенту (диагностика, поиск ошибок)
- управлять доступами и поддержкой без входа в «продуктовую» админку клиента

## 2) Принципы безопасности
- Operator Console — отдельное приложение/раздел с отдельной ролью **SuperAdmin**
- Жёсткое логирование действий (audit trail)
- Имперсонация (вход «как пользователь») только:
  - по явному действию
  - с причиной
  - с ограничением по времени
  - с логированием всех действий

## 3) Основные разделы
### Tenants
- список компаний
- статус: active / trial / suspended
- тариф, количество пользователей/объектов
- лимиты (storage, seats)

### Users
- пользователи по tenant
- сброс MFA/пароля (если используете)
- блокировка/разблокировка

### Billing
- подписки
- платежи
- счета
- возвраты
- webhooks status

### Support / Diagnostics
- поиск по ошибкам (Sentry), логам
- журнал событий (audit log)
- просмотр интеграций (storage, email)

### Feature Flags
- включение/выключение функций по tenant
- beta-функции

### Content / i18n (позже)
- управление переводами, шаблонами писем, текстами

## 4) Operator роли
- SuperAdmin (полный доступ)
- Support Agent (без биллинга/удалений)
- Billing Admin (платежи/подписки)
- Read-only Auditor

## 5) Техническая реализация (вариант)
- Отдельный Next.js app (`/apps/operator`)
- Отдельные маршруты и middleware
- Отдельные таблицы: OperatorUser, OperatorRole
- Доступ к данным tenants через сервисный слой

## 6) MVP Operator Console
- список tenants
- просмотр пользователей tenant
- просмотр подписки/платежей
- impersonation (минимальная)
- audit log
