# BuildOS (Польша) — рабочий холст
RU/PL • Multi-tenant SaaS для строительных компаний

## 0) Цель продукта
Единая система для строительной компании: сметы/калькуляция, управление объектами и этапами работ, бригады/субподряд, финансы, документы, фото/прогресс, клиентский кабинет, (позже) визуализация/планировка.

**Стартовый рынок:** Польша. Языки интерфейса: RU + PL.

## 1) Аудитория и модель tenancy
- **Tenant = строительная компания (фирма)**
- Внутри Tenant: пользователи, роли, прайсы, объекты, сметы, документы, материалы, бригады.
- Изоляция данных: `tenant_id` в каждой сущности.

## 2) Роли (первичный список) и кому что нужно
> Принцип: роли = набор прав. Один человек может иметь несколько ролей.

### Core роли
- **Owner (Владелец фирмы)** — видит всё: маржинальность, финансы, отчёты, настройки.
- **Admin (Админ)** — пользователи/права/справочники.
- **Project Manager (Менеджер проекта)** — объекты, сроки, этапы, сметы, коммуникация.
- **Site Manager / Foreman (Прораб)** — задачи по объекту, бригады, факты выполнения, фотоотчёты.
- **Worker (Работник)** — свои задачи, отчёт, время, фото.
- **Client (Клиент)** — доступ к КП/смете, статусам этапов, фото, платежам.

### Финансы/документы
- **Accountant (Бухгалтер)** — счета/акты/платежи/экспорт.
- **Finance Controller (Финансист)** — бюджеты, cashflow, прибыльность по объектам.

### Проектирование/дизайн
- **Designer (Дизайнер интерьера)** — визуальные материалы, ТЗ, подбор материалов.
- **Architect (Архитектор)** — документация, планы, версии.

### Предложения: кого ещё добавить
- **Estimator / Quantity Surveyor (Сметчик)** — правила расчёта, нормы расхода, прайсы.
- **Procurement / Purchaser (Закупщик)** — поставщики, заказы материалов, цены, доставка.
- **Warehouse / Storekeeper (Кладовщик)** — остатки, выдача, списания.
- **Inspector / QA (Технадзор/Контроль качества)** — чек-листы, приёмка этапов, дефекты.
- **HSE / Safety (Охрана труда)** — инструкции, инциденты, допуски.
- **Subcontractor (Субподрядчик)** — доступ к своему объёму работ/актам/оплатам.
- **Sales / CRM (Менеджер продаж)** — лиды, КП, конверсия (можно позже).

## 5) Смета и калькуляция (ядро продукта)
### Две цены
- **Цена для клиента** (revenue)
- **Себестоимость** (subcontract + materials + labor + overhead)
- Маржа и прибыль по объекту/этапу.

### Что нужно в расчёте
- Площадь/периметр/высота (в зависимости от работ)
- Нормы расхода материалов
- Прайс работ (за м²/пог.м/точку)
- Коэффициенты: сложность, срочность, состояние, этажность, доступность
- Минимальные суммы, округления
- Версии сметы (v1/v2) и сравнение дельт

## 6) Фото/прогресс и доказательства выполнения
- Фото **до** / **в процессе** / **после**
- Привязка фото к этапу/помещению/задаче
- Таймлайн объекта
- Комментарии, отметки дефектов

## 7) Клиентский кабинет
- КП/смета (PDF + веб-версия)
- Статусы этапов и прогноз завершения
- Фотоотчёты
- Счета/платежи
- Коммуникация (комментарии/запросы)

## 8) Модули продукта (по приоритету)
### MVP (первая продаваемая версия)
1. Multi-tenant + пользователи/роли
2. Объекты + помещения
3. Шаблоны этапов + статусы
4. Калькулятор сметы (клиент/себестоимость) + версии
5. Материалы (каталог + нормы расхода — упрощённо)
6. Фото-лог + таймлайн
7. Экспорт PDF (КП/смета)

### Phase 2
- Закупки/склад
- Субподрядчики: акты, выплаты
- Планирование бригад и календарь
- Финансы: cashflow, бюджеты
- Интеграции (Google Drive, e-mail, WhatsApp)

### Phase 3
- Визуализация помещений (2D план, замеры, зоны)
- BI/дашборды, прогнозы, LTV клиента

## 9) Роадмап (черновик)
### Этап 0 — Подготовка (1–3 дня)
- PRD v0 (этот документ)
- RBAC матрица (права по ролям)
- Модель данных v0 (таблицы)
- Карта экранов MVP
- Backlog + шаблоны задач

### Этап 1 — Основа платформы (1–2 недели)
- Auth + tenancy
- Users/Roles
- CRUD: Companies / Objects / Rooms

### Этап 2 — Смета (2–4 недели)
- Price books (works/materials)
- Estimate engine (формулы)
- PDF export
- Маржинальность

### Этап 3 — Этапы и фото (2 недели)
- Stage workflow
- Upload + timeline
- Комментарии/дефекты

### Этап 4 — Подготовка к продаже (1–2 недели)
- Онбординг
- Мини-биллинг (планы/оплаты можно позже)
- Логи/аудит

## Приложение A — идеи для «killer features»
- Шаблоны объектов (квартира 40/50/60м²) + расчёт за 2 минуты
- Версионность сметы с сравнением дельт по марже
- Фото-отчёт по этапам как доказательство для клиента
- Приёмка этапов по чек-листам (QA)
